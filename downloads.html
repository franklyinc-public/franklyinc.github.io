iOS SDK v1.2.0 Released

August 3, 2015
Changes

  1. Simplified approach to authentication on the client which makes it easier to handle support for Guest Users and upgrading Guest Users to Secured Users.
    • identifierWithNonce:preferredRole:completion:'s method signature has changed to identityTokenWithNonce:preferredRole:completion:
    • identityTokenWithNonce:preferredRole:completion's completion handler now expects to be passed your with userId as the second parameter. This is so we can properly determine whether or not to re-use stored session credentials in the future.
    • initiateAuthenticationWithNewSession: has been deprecated in favor of initiateAuthenticationWithUserID:role: to provide us with a better mechanism for caching session credentials.
    • logOut has been added to FranklySDK. logOut will clear any cached session credentials and disconnect from Frankly real-time services.

New Features

  1. Announcers
    • You can specify people in a Room that can send Sticky Messages.
    • Messages sent by Announcers will be easily identifiable and they are able to send both text and photo Sticky Messages.
    • Keep on the look out for updates to the Admin Console and our Server SDKs for more information on how to add a user as an Announcer.
  2. Conditional Rooms
    • Via the Frankly Admin Console, define a set of conditions that your users can identify with in order to customize which Rooms they see.
      • indexViewController and indexNavigationController have been added to FranklySDK. These provide access to Conditional Rooms.
      • These will replace roomListViewController and roomListNavigationController in the near future. Sign up for email updates above to be notified of a more concrete timeline.
  3. Push Notification Support
    • Now users can receive Push Notifications for any Sticky Messages sent in Rooms they've subscribed to.
      • registerForNotificationsWithDeviceToken has been added to FranklySDK. You may pass either the concatenated device token string or the NSData object you are passed in application:didRegisterForRemoteNotificationsWithDeviceToken:. Registering with the Frankly servers will allow users to receive notifications when new Sticky Messages are posted in Rooms they are subscribed to.
      • If you you would like to make use of this feature, please reach out to your Frankly contact to facilitate adding this to your account. Our plan is to add this support to the Franky Admin Console in the near future.
      • handleNotification: has been added to FranklySDK. You may pass your entire notification payload to this method, which FranklySDK will parse to determine if it has a relevant view and return it. Currently, this will return either the Room view associated with the notification payload or nil if there is no Frankly slug in the payload.
  4. Tappable URLs
    • URLs are now hyperlinked.
