Before Implementing the Frankly SDK

  1. You’ll need to obtain an appKey and appSecret before implementing the Frankly SDK. These are required for authenticating with our server. If you don’t have these, please contact us at platform-support@franklychat.com.
  2. Your server will need to be able to generate a JSON Web Token (JWT). To learn more about how to generate a JWT, click here.

Basic Concepts

At the center of Frankly’s conceptual model is the Room, where chat happens. Rooms belong to Roomlists, and Users can subscribe to Rooms.

Inside of a room, a user can send three kinds of messages depending on their role:

  • A standard chat message.
  • An Announcement, which is sent by the brand.
  • A Sticky Message, a message sent by the brand which pins to the top of the room once it reaches the top of the screen.
    • Sticky Messages are shown with the brand’s App Avatar.
    • New Sticky Messages will replace the last pinned Sticky Message after reaching the top.

Installing with NPM

The recommended way to access the Frankly JavaScript SDK is via the public NPM repo named frankly-js.

If preferred, you can also download the SDK via the public Frankly github repo.

To install via NPM:

1
npm install frankly-js

Initializing the JavaScript SDK

All operations to the API are made from instances of the FranklyClient class. Those objects expose methods to the application which map to remote procedure calls (RPCs) on the Frankly servers and they also negotiate and maintain the state required by the API’s security policies.

Here’s how FranklyClient instances are created:

1
2
            var FranklyClient = require( 'frankly-js' )
            var client = new FranklyClient()
