Getting Started

Welcome to the Frankly Developer Portal. Here you'll find all the guidance necessary in order to download Frankly's SDKs and communicate with Frankly's server APIs to integrate chat into your application's experience. We're constantly making improvements, so check back often for updates.

Signing Up for Access

Note that in order to integrate the Frankly SDK or communicate with Frankly servers, you'll need an Admin Console account where you can get an appKey and appSecret, as well as manage your application's settings.


Interested in getting an Admin Console account to integrate with the Frankly platform? Contact us at platform-support@franklychat.com to find out how your brand can engage its users with the power of chat.

Concepts

Coming soon - a breakdown of Frankly-specific reference to the various features you'll encounter when using and integrating with the Frankly platform.


Rooms

An individual chat room based around a certain topic.


Room List

The list of Rooms available to a user.


Subscriptions

Allow users to bookmark a specific Room to the top of the Room List. Subscribing turns on notifications for brand messages (see 'Announcements' and 'Sticky Messages' below) and automatically pulls the Room's history for a more seamless experience when revisiting the Room.


Chat Messages

Messages sent by users in a Room.


Announcements

A message sent by the brand


Sticky Messages

A message sent by the brand to be seen in-line with all other chat messages and is pinned to the top of a Room once it reaches the top edge of the screen. Sticky Messages are shown with the brand's App Avatar and new Sticky Messages will replace the last pinned Sticky Message after reaching the top.


Sticky Message Filter

A toggle option for a user to see a list of all Sticky Messages sent by a brand.


Input Bar

The area where a user types their message to be sent.


Stickers

Icons that can be sent as a message. They are available in the Input Bar by selecting the sticker icon to the left of "Type a message." When a sticker is first clicked, is it selected. When it is clicked a second time, it is sent as a message to the Room.


Authentication

The status of a user in a Room.

  • Spectator Mode
    • User is allowed to view all messages in a Room.
    • User is allowed to view all rooms in the room list, unless otherwise determined by an administrator.
    • User cannot send messages or Subscribe to a Room.
  • Guest Mode
    • User picks a display name to be shown beside their messages.
    • User is allowed to view all messages in a Room.
    • User is allowed to view all Rooms in the Room List, unless otherwise determined by an administrator.
    • User is allowed to send messages and subscribe to a Room.
    • User's account is not secured/cannot be used to sign in on a different device.
  • Authenticated
    • User provides authentication information to the brand's application.
    • User is allowed to view all messages in a Room.
    • User is allowed to view all Rooms in the Room List.
    • User is allowed to send messages and subscribe to a Room.
    • User's account history is secured with a username and password, collected by the partner.
    • User is able to edit their Display Name and Profile Avatar in the Profile Card.

Profile Card

The area where a user can edit their Display Name and Profile Avatar.


Admin Console

Dashboard where a brand can do the following:

  • Add, edit, or delete Rooms.
  • Send Announcements and Sticky Messages.
  • Moderate users and messages.
  • Assign administration roles.
