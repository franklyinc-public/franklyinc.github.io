Setting Up Your First Room List

Important Note Please make sure you've set up at least one Room in the Admin Console before proceeding. If you're unsure of how to do this or if you don't have Admin Console access, please see Getting Started for next steps.


Calling the Room List via the iOS SDK


To call the Room List, retrieve the room list controller, [FranklySDK roomListViewController], and present it in your app or push it on your navigation controller.



Calling the Room List via the Android SDK

The Frankly SDK contains a RoomListActivity class which contains everything needed to show the chatroom list and allow the user to join chatrooms: 


Intent intent = new Intent(context, RoomListActivity.class);

context.startActivity(intent);

Setting the Display Name & Profile Avatar

A couple thigns to note about these setting these properties for a user's Frankly profile:

  • The maximum length of a Frankly Display Name is 18 characters
  • For best results, we recommend uploaded Profile Avatars to be 250x250px compressed JPGs
  • Important Note The Frankly Chat Platform automatically provides default Profile Avatars for users where it has not been set for them, to provide a nicer out-of-the-box experience

Setting User Info via the iOS SDK

Set the FranklySDK’s profileDelegate to an object that conforms to the FranklySDKUserProfileDelegate protocol. In our example code, we’ve made our ViewController also conform to this protocol: 



@interface ViewController : UIViewController <FranklySDKAuthenticationDelegate, FranklySDKUserProfileDelegate>



- (void)viewDidLoad {

	[super viewDidLoad];

	[FranklySDK sharedInstance].authenticationDelegate = self;
	[FranklySDK sharedInstance].profileDelegate = self;				

	[[FranklySDK sharedInstance] initiateAuthentication];
}

The profile delegate will get called every time the user profile image or display name changes. Also, you can enable or disable user editing of their profile data within the FranklySDK UI by allowing your delegate respond appropriately to the -(BOOL)shouldAllowProfileEditing delegate request.


Setting the user's UserName and profileImage can be done via the calls through the FranklySDK object: 


-(void)setUserName:(NSString *)userName profileImage:(UIImage *)image];

When the user's UserName and profileImage have been updated and synced with Frankly servers, the profileDelegate will get the callbacks didUpdateDisplayName and didUpdateProfileImage, confirming that these properties have been set. At the moment, the profileDelegate does not return a failure callback, so please handle accordingly. Please also keep in mind, that in the event that you want to set UserName or profileImage, you can pass in nil for whichever property you do not wish to update. Also note that the call to set these properties does nothing if called before receiving authenticationSucceeded delegate message.

Setting User Info via the Android SDK

Developers can upload new Profile Avatars for users to the Frankly servers where the Intent contains either the data URI of the Intent, or an extra called "data" which stores a Bitmap (the Android Gallery app currently returns the data in this format in onActivityResult()) as follows: 


final UserManager userManager = UserManager.getInstance(context);
final String userAvatarUrlString = userManager.uploadUserAvatar(context, intent);

Note that simply uploading the image does not tie the image to any particular user. The returned URL string must be set on a User object and passed to UserManager for persisting as follows: 


user.setAvatarImageUrlString(userAvatarUrlString);
userManager.update(user);

Similarly, to update the display name of a user, simply do the following: 


user.setDisplayName(displayName);
userManager.update(user);

WARNING: All UserManager calls must be done on a non-UI thread.

Embedding a Room on Web


The Frankly Web App can be embedded directly into your web app through the use of JavaScript and iframes. This service allows your users the ability to use Frankly supported chat without having to leave your web app.

There are two steps that need to be taken in order to embed Frankly within your web app:

  • Include the Frankly Widget script on the page that will contain the Frankly Web App, including a small script on the page to initialize the Frankly Web App.
  • Embed the Frankly Web App using the <iframe> tag.

More detail on these steps below.






1. Include Frankly Widget Script


Include the following line of code in the head of your HTML: 


<script src="//d13r8avoofccpt.cloudfront.net/0.2.1/frankly.min.js"></script>
<script>
  frankly.widgets.init({
      onAuthenticationRequest: function(nonce){
        console.log(nonce);
        return new Promise(function (resolve, reject) {
            /*
             * TODO: generate an identity token and pass it to resolve, or call
             * reject with an error.
             */
        });
      }
  });
</script>

This script sets the configuration for your page to host the Frankly Web App, allowing it to run within your web app.

The call to frankly.widgets.init configures the web page to properly communicate with the embedded Frankly Widgets.

The function set to onAuthenticationRequest will be called when the widget needs to authenticate as a user in order to render the view and enable chatting. It must return a javascript promise where the resolve callback must be called with the generated identity token.

Note the section of the above script that includes the TODO statement. This section must be replaced with code that generates an identity token using one of Frankly’s server SDKs. We strongly recommend that you generate this token on your server side, and call it from a script within this script block. Exposing your Frankly App key and secret within this code block may compromise the security of your Frankly Web App. For more information on generating an identity token using one of Frankly’s server SDKs, please consult the documentation on generating identity tokens.


2. Embed the Frankly Web App in an iframe


Include the following line of code in the body of your HTML, at the place that you want the Frankly Web App to appear: 


	<iframe src="https://app.franklychat.com/web/{app_id}/#/{room_id}"></iframe>

You will need to replace {app_id} with your app’s unique numeric ID, e.g. 123, and replace {room_id} with your room’s unique numeric ID, e.g. 9876. If you wish to display all room avatar tiles to your users, rather than embedding a specific room, then include the following code in your HTML: 


	<iframe src="https://app.franklychat.com/web/{app_id}"></iframe>

You can get your app_id by logging into the Frankly Admin Panel and selecting Settings in the left hand menu.

Customizing UI in the SDK

This section will describe how the Frankly SDKs allow you to customize various components of the provided UI.

Customizing the iOS SDK

FranklySDK on iOS allows partners to set values that affect FranklySDK's interface within the partner app. This data is passed in via a standard plist file. An example data file, FRSDKExampleConfig.plist, is included FranklySDK package.

Partners may modify that file, or create a new one and then, any time after initializeSDKWithServerType: has been called,add the file to FranklySDK via the function addCustomizationFile:(NSURL *)fileURL. Multiple files may be added via this call, any collisions with existing keys will cause the existing value to be overwritten by the new value.


Customization Plist

This is a standard Apple plist file which means it’s an XML file with a single object named Root. FranklySDK expects this Root object to be a dictionary of dictionaries. Each dictionary represents a customization category that is populated with key value pairs.


Data Types

The base values stored in the dictionaries in the customization file will be of the following types:

  • Data Types This is a string that follows a standard hex string format with alpha. It maybe be preceded by ‘0x’ or ‘#’ but the last 8 characters should represent the hex values in red, green, blue, alpha order. So the color white with full opacity would be represented as 0xFFFFFFFF and black, haf opacity would be represented as 0x0000007F.

Customization Categories
  • ui
    • the ui key contains a dictionary of key value pairs with the following keys:

Color Key Use Case
primary-fg-color Takes a color value and is used throughout the app as primary foreground color.
primary-bg-color Takes a color value and is used throughout the app as the primary background color.
accent-bg-color Takes a color value and is used throughout the app as an accent color. Most noticeably as the background of the locked Input Bar.
brand-bg-color Takes a color value. Is used as the background for Sticky Messages when they are docked at the top of a Room.
attention-bg-color Takes a color value. Is used in the custom activity indicators.

Customizing the Android SDK

To assist with allowing apps to provide their own basic customizations to existing Frankly SDK Views, Activities, and Fragments, you can override the default styles in XML style files: 


<?xml version="1.0" encoding="utf-8"?>
<resources>
    <color name="frankly_sdk_primary_background">#151515</color>
    <color name="frankly_sdk_primary_foreground">#FFFFFF</color>
    <color name="frankly_sdk_accent_background">#FD3C1C</color>
    <color name="frankly_sdk_accent_foreground">#FFFFFF</color>
    <color name="frankly_sdk_brand_background">#252525</color>
    <color name="frankly_sdk_brand_foreground">#FFFFFF</color>
    <color name="frankly_sdk_attention">#4891F0</color>
</resources>


Color Key Use Case
frankly_sdk_primary_background Activity actionbar background
frankly_sdk_primary_foreground Activity actionbar foreground
frankly_sdk_accent_background Indicating a primed state such as send button active state, toggle checked, subscribed room, locked bar background for input message bar
frankly_sdk_accent_foreground Foreground items color where the background has frankly_sdk_accent_background color such as locked bar forground items
frankly_sdk_brand_background Denoting who and what explicitly belongs to the brand such as sticky messages cell background
frankly_sdk_brand_foreground Foreground items color where the background has frankly_sdk_brand_background color such as sticky messages foreground items
frankly_sdk_attention Indicating an attention to user such as new sticker pack indicator and activity indicator base color
