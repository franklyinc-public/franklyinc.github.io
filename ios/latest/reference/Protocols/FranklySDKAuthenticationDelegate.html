FranklySDKAuthenticationDelegate Protocol

– identityTokenWithNonce:preferredRole:completion: required method

Frankly SDK makes this call to request the partner retrieve a valid identifer from the partner’s server based on the nonce received from Frankly’s server

- (void)identityTokenWithNonce:(NSString *)nonce preferredRole:(FRSDKAuthenticationRole)preferredRole completion:(void ( ^ ) ( NSString *identityToken , NSString *userId ))completion

Parameters

nonce

the one time use string retrieved from Frankly’s servers by the SDK
completion

finishes the authentication process when the partner has retrieved a valid identifier based on the nonce from their backend.

Discussion

Note: the nonce is only good for a single authentication and it will also expire if not used within a short period of time.

Frankly SDK calls this to let the partner app know that Frankly’s SDK is initialized and may be used

- (void)authenticationDidSucceedWithUserID:(NSNumber *)userID

Parameters

userID

the id FranklySDK uses to reference this user. It will NOT be the same id submitted in the completion block of identityTokenWithNonce:completion:. The userID will never be null but an anonymous user will have a valid userID with an integer value of 0.

– doUserAuthentication required method

notifies the delegate that the user needs to sign in or sign up.

- (void)doUserAuthentication

Discussion

The delegate is responsible for presenting the user a way to log in, sign up, or authenticate either via a or login screen or via some other mechanism. The delegate is also responsible for validating the authentication.

Note: After properly validating the user, call initiateAuthentication to begin the process of connecting to Frankly’s chat service.
